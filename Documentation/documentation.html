Installation & Deployment

Requirements

  • A computer with any recent Windows, macOS or Linux, and Internet access
  • Node.js and npm: You can learn more about it and download for free here (installers available here) OR yarn, also available for free here

First steps

  • Unzip "portfolio-template.zip" files in a folder of your choice (example: "C:/website/")
  • Open a CLI (by example on Windows, use cmd.exe) in the folder where you unziped the files (example: "C:/website/"), you will need to type the commands below, be sure to be in the right folder
  • Install the dependencies, command: npm install or yarn
Deploy development server (for testing) on your local machine

Start development server, so the page on your web browser auto-updates as you edit the files

  • Run the local web server with nextjs, command: npm run dev or yarn dev
  • You should be able to access your website on this default URL (check the logs in the CLI):
    http://localhost:3000

Deploy online (export static pages)

You can export your project to static pages, so you can simply copy/paste the files to any web server and your website is ready!

  • Build the project, command: npm run build or yarn build
  • Generate the pages, command: npm run export or yarn export (by default it'll generate them in "out/" folder)
  • Copy the files of the "out/" folder to the root folder of your website's server. By example if your domain my.domain.com points to your server's folder /my/server/www/, put the files in that /my/server/www/ folder
You will have to repeat those steps if you edit the projects files and want to update your static pages. DO NOT directly edit the static pages, because they will be overridden when you do a new export.

Deploy online with Vercel

  • Create a Vercel account, it's free, you can register with your email/GitHub/GitLab/Bitbucket account here
  • Install Vercel CLI, command: npm i -g vercel or yarn global add vercel
  • Run Vercel in the root folder of your project (example: "C:/website/"), command: vercel
  • Simply follow the instructions, if it's the first time you use it, you'll have to login with Vercel account (it's done in the CLI). The project name you choose will be the first part of the Vercel's website URL (it's not important if you want to use your own domain name). You can link it to any existing project, else it'll create it automatically for you. It'll also ask you if you want to override the settings, you can refuse by replying "n"
  • Then it'll deploy your project in production on Vercel servers (example: https://myprojectname.vercel.app)
  • To use your own domain name, go to your project settings on your Vercel's dashboard (example: https://vercel.com/myusername/myprojectname/settings)

To override your production website, type: vercel --prod, else type vercel to deploy a "Preview" version which is aslo accessible online. You can also change any "Preview" version to production on Vercel's dashboard.
More info about Vercel's deployment here.

(Optional) Deploy online from you Git with Vercel

Vercel allows for automatic deployments of every push of your GitHub, GitLab, and Bitbucket projects, more info here.

(Optional) Deploy online on your server or Docker

You can deploy to any hosting provider that supports Node.js, and also on a Docker container, more info here. But we recommend you to use Vercel, as it is free and zero configuration needed.

NOTE:

On Windows, there might be a bug after the first deployment with Vercel, the CLI window might freeze, you can simply close it.

How To Edit Content & Colors

Most of the content can be found in "/content" and "/components" folders, here a non-exhaustive list:

  • "pages/index.tsx" is the main page.
  • "pages/404.tsx" is the 404 error page.
  • "components/Footer.tsx" is the footer (really?).
  • "components/Socials.tsx" is a list of your social network profiles.
  • "/content/skills.ts" is a list of tech/skills that you use with their corresponding icons from Iconfiy.
  • "/content/experiences.ts" is a list of your work history in a resume/CV format.
  • "/content/posts.ts" is a list of blog posts.
  • "/content/projects.tsx" is a list of projects you've worked in, the tech you used, and some images to showcase your work.
  • "/content/services.ts" is a list of services you offer to clients.

Make sure to also modify for SEO:

  • "/components/SiteLayout.tsx"

Edit this file to change the main colors:

  • "tailwind.config.js"

For project and blog images, stick with a 4x3 ratio.
